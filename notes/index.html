Notes on building AI you can actually trust.

Essays and build notes on designing AI systems for regulated and high-complexity work. Each note sits on top of something I have built, so the argument is anchored to a working tool rather than a slide. If you want the reasoning behind the practice, this is where it lives.

What this is

Reasoning you can check against a build

Most writing about AI stays at the level of the prediction. These notes stay at the level of the mechanism: why a system fails on the exact thing that matters, and what design choice closes the gap. Each essay links to the tool it is anchored to, so you can read the argument and then run the thing it describes.
